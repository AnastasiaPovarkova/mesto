## Автоматизация тестирования (QA Automation)

Проект покрыт автоматическими end-to-end (E2E) тестами пользовательского интерфейса.

**Стек автоматизации:**
* **Python 3** — основной язык разработки тестов.
* **PyTest** — фреймворк для сборки и запуска тестовых сессий.
* **Selenium WebDriver** — управление браузером и симуляция действий пользователя.
* **Page Object Model (POM)** — архитектурный паттерн для разделения логики страниц и логики тестов.
* **Allure Framework** — построение подробных интерактивных отчетов о прохождении тестов.
* **GitHub Actions** — настроен CI/CD пайплайн для автоматического запуска тестов в headless-режиме при каждом пуше.

### Покрытие тестами:
1. **Профиль пользователя:** проверка двойного редактирования данных (имя/профессия) с контролем ожидания синхронизации данных с бэкендом.
2. **Карточки мест:** полный жизненный цикл (открытие попапа, создание новой карточки, проверка корректности добавления в топ списка, установка лайка с обновлением счетчика и последующее удаление карточки для очистки данных).

## Команды для сборки и запуска проекта

В папке проекта вы можете запустить команды:

```bash
npm run dev
```
Запустить проект в режиме разработки.
Страница будет автоматически обновляться при изменении кода, ошибки отслеживаются в консоли.
Адрес страницы: http://localhost:8080/

```bash
npm run predeploy
```
Скрипт сгенерирует оптимизированную сборку проекта. Внутри проекта появится новая папка `build`. Внутри `build/static` можно найти оптимизированные версии всего написанного кода, наряду с другими ресурсами: JS, CSS и шрифтами. 

Затем эти файлы можно разместить на любом сервере, т.е. использовать для деплоя веб-приложения в интернете.

### Запуск автотестов локально

```bash
cd tests_automation
# Запуск тестов с генерацией результатов
pytest --alluredir=allure-results

# Открытие интерактивного отчета Allure в браузере
allure serve allure-results
```
